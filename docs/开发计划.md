# LeadMine 开发计划

## 当前交付摘要

- **当前主线状态**：微信公众号 RSSHub 数据源支持、线索当前页批量操作、线索页初始关键词加载修复、数据处理/来源类型相关回归测试增强均已交付。
- **文档状态**：规划与产品文档已完成整理，统一归档到 `docs/` 与 `docs/superpowers/`。
- **本阶段重点**：以交付质量基线为核心，确认当前已交付能力的最小验证闭环，而非继续扩展新功能范围。
- **基线验证命令**：`cd backend && python3 -m pytest`、`cd web && npm test`

## 1. 项目概述

- **项目名称**：LeadMine（线索矿工）
- **项目类型**：B2B销售线索挖掘系统
- **技术栈**：Python + FastAPI + Vue3 + MySQL + Elasticsearch + Docker
- **预计周期**：6周

## 6. 风险评估

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| 新闻网站反爬 | 高 | 高 | 使用代理IP、控制频率、备用源 |
| 微信公众号采集困难 | 高 | 高 | RSSHub优先、备选方案 |
| 线索准确率低 | 中 | 中 | 人工复核、优化规则 |
| 开发延期 | 中 | 中 | 预留缓冲、优先级排序 |
| 服务器成本超支 | 低 | 低 | 选择合适配置 |
