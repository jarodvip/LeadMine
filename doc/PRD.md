# LeadMine 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品背景

在当今信息爆炸的时代，销售团队面临着海量的商业信息挑战。每天从新闻网站、微信公众号、RSS订阅、社交媒体等渠道产生的商业资讯数量巨大，其中蕴含着大量潜在的销售线索。然而，人工筛选这些信息效率低下，容易遗漏重要商机。

### 1.2 产品定位

LeadMine（线索矿工）是一款**热点新闻采集与销售线索挖掘系统**，旨在帮助销售团队从多渠道信息中自动识别、提取有价值的商业线索，实现销售情报的自动化收集与整理。

### 1.3 目标用户

- B2B销售团队
- 商务拓展（BD）部门
- 市场情报分析团队
- 投资分析师

---

## 2. 产品目标

### 2.1 核心目标

1. **多渠道数据采集**：实时抓取新闻网站、微信公众号、RSS订阅、社交媒体等渠道的热点资讯
2. **智能数据清洗**：自动去重、过滤、分类，提取高质量内容
3. **线索智能识别**：从新闻中自动识别融资、并购、产品发布、招标采购等销售线索
4. **企业信息补充**：基于公开渠道补充企业基本信息
5. **高效线索管理**：提供Web后台进行线索查看、筛选、分配、跟进

### 2.2 成功指标

| 指标 | 目标值 |
|------|--------|
| 日均采集新闻量 | 500-2000条 |
| 线索识别准确率 | ≥70% |
| 系统可用性 | ≥99% |
| 线索转化率 | ≥5% |

## 3. 功能需求

### 3.1 数据采集模块

#### 3.1.1 新闻网站采集

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| 36氪爬虫 | P0 | 抓取36氪科技创业新闻 |
| 虎嗅爬虫 | P0 | 抓取虎嗅商业科技文章 |
| 钛媒体爬虫 | P1 | 抓取钛媒体TMT行业新闻 |
| 创业邦爬虫 | P1 | 抓取创业邦融资动态 |
| 自定义爬虫 | P2 | 支持用户配置新的新闻源 |

#### 3.1.2 RSS订阅

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| RSS源管理 | P0 | 添加、编辑、删除RSS订阅源 |
| 定时抓取 | P0 | 按配置的频率自动抓取RSS内容 |
| RSSHub集成 | P1 | 支持RSSHub将微信公众号、微博等转为RSS |

#### 3.1.3 社交媒体

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| 微博采集 | P1 | 抓取企业官方微博内容 |
| 知乎采集 | P1 | 抓取行业话题和热门回答 |
| 雪球采集 | P2 | 抓取财经讨论和个股资讯 |

### 3.2 数据处理模块

#### 3.2.1 数据清洗

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| 编码统一 | P0 | 将所有内容统一转换为UTF-8编码 |
| HTML清洗 | P0 | 去除HTML标签，提取纯文本 |
| 文本提取 | P0 | 使用readability算法提取正文内容 |
| 去重处理 | P0 | 基于SimHash算法去除重复内容 |
| 关键词过滤 | P1 | 根据预设关键词过滤无关内容 |

#### 3.2.2 NLP处理

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| 中文分词 | P0 | 使用jieba进行中文分词 |
| 关键词提取 | P0 | 提取文章关键词 |
| 实体识别 | P0 | 识别企业名称、人名、产品名等 |
| 文本分类 | P1 | 将文章分类到预设的行业中 |

| 招标采购 | 招标、采购、供应商招募、寻求合作 | X公司招标云服务商 |

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 系统响应时间 | 页面加载 ≤2秒，API响应 ≤500ms |
| 并发用户数 | 支持10-50人同时使用 |
| 数据处理能力 | 日处理新闻 ≥2000条 |

### 4.2 可用性需求

| 指标 | 要求 |
|------|------|
| 系统可用性 | ≥99% |
| 数据采集成功率 | ≥95% |
| 故障恢复时间 | ≤30分钟 |

### 4.3 扩展性需求

- 支持快速添加新的新闻源
- 支持横向扩展以应对更大数据量
- 模块化设计，便于功能升级

### 4.4 安全需求

- 用户身份认证
- 操作日志记录
- 敏感数据脱敏

## 5. 产品路线图

### Phase 1：基础能力建设（1-2周）

- 基础框架搭建
- RSS采集功能
- 核心新闻网站爬虫（36氪、虎嗅）

### Phase 2：数据处理能力（2周）

- 数据清洗流程完善
- NLP处理能力
- 线索识别规则

### Phase 3：产品化完善（2周）

- Web管理后台
- 线索管理功能
- 通知和导出

### Phase 4：持续优化（持续）

- 更多数据源接入
- 识别准确率优化
- 用户反馈迭代

## 6. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 新闻网站反爬 | 高 | 使用代理IP池、控制抓取频率 |
| 微信公众号采集困难 | 高 | 使用RSSHub方案或第三方接口 |
| 线索误识别 | 中 | 人工复核机制+规则优化 |
| 数据存储成本 | 中 | 定期清理过期数据 |

## 7. 术语表

| 术语 | 定义 |
|------|------|
| 线索 | 从新闻中识别出的商业信息，如融资、并购等 |
| 去重 | 识别并去除重复或相似的内容 |
| 实体识别 | 从文本中识别出人名、地名、企业名等 |
| RSS | Really Simple Syndication，简易信息聚合 |
| RSSHub | 开源RSS生成器，可将各种网站转为RSS |
